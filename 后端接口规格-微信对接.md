## 接口一：微信小程序登录

### 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/wx-login` |
| 鉴权 | 无需 |
| 用途 | 接收小程序 wx.login 返回的临时 code，换取用户 openid 并签发 JWT |

### 请求体

```json
{
  "code": "0a3x1A000abc..."
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `code` | string | ✅ | `wx.login()` 返回的临时登录凭证（有效期 5 分钟，仅可使用一次） |

### 后端内部调用流程

```
1. 调用微信 code2Session 接口：
   GET https://api.weixin.qq.com/sns/jscode2session
   ?appid=wx8cf5c2ef49e867fe
   &secret={小程序的AppSecret，在微信公众平台获取}
   &js_code={客户端传来的 code}
   &grant_type=authorization_code
```

返回示例：
```json
{
  "openid": "oGZUI0eg********",
  "session_key": "HyVFkGlu****",
  "unionid": "o6_bmas****",
  "errcode": 0,
  "errmsg": "ok"
}
```

```
2. 根据 openid 查找已有用户，不存在则创建新用户记录
3. 将 session_key 安全存储在服务端（不要下发给客户端）
4. 生成 JWT token（包含 openid、userId，有效期建议 7 天）
```

### 响应体

```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "openid": "oGZUI0eg********",
  "unionid": "o6_bmas****"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `token` | string | 用户登录 JWT |
| `openid` | string | 微信 openid（用户唯一标识） |
| `unionid` | string | 微信 unionid（可选） |

### 错误响应

```json
{
  "errcode": 40029,
  "errmsg": "code 无效或已过期"
}
```

## 接口二：手机号解密

### 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/phone` |
| 鉴权 | Bearer Token（ERP 管理员 JWT） |
| 用途 | 接收小程序 getPhoneNumber 返回的 code，解密为用户手机号 |

### 请求体

```json
{
  "code": "8b2c3d4e5f..."
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `code` | string | ✅ | `<button open-type="getPhoneNumber">` 回调中 `e.detail.code`（有效期 5 分钟，仅可使用一次） |

### 后端内部调用流程

```
1. 获取 access_token（微信全局接口调用凭据，有效期 7200 秒，需缓存）：
   GET https://api.weixin.qq.com/cgi-bin/token
   ?grant_type=client_credential
   &appid=wx8cf5c2ef49e867fe
   &secret={小程序 AppSecret}
```

返回：
```json
{
  "access_token": "55_xxxxx...",
  "expires_in": 7200
}
```

```
2. 用 access_token 调用手机号解密接口：
   POST https://api.weixin.qq.com/wxa/business/getuserphonenumber
   ?access_token={ACCESS_TOKEN}
   Content-Type: application/json
   
   Body:
   {
     "code": "客户端传来的 code"
   }
```

微信返回：
```json
{
  "errcode": 0,
  "errmsg": "ok",
  "phone_info": {
    "phoneNumber": "1381234****",
    "purePhoneNumber": "1381234****",
    "countryCode": "86",
    "watermark": {
      "timestamp": 1234567890,
      "appid": "wx8cf5c2ef49e867fe"
    }
  }
}
```

### 响应体

```json
{
  "phoneNumber": "1381234****",
  "purePhoneNumber": "1381234****",
  "countryCode": "86"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `phoneNumber` | string | 带区号的手机号 |
| `purePhoneNumber` | string | 纯数字手机号 |
| `countryCode` | string | 国家代码（如 86） |

## 接口三：微信支付统一下单

### 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/payment/unify` |
| 鉴权 | Bearer Token（ERP 管理员 JWT） |
| 用途 | 创建微信支付订单，返回小程序调起支付所需的签名参数 |

### 请求体

```json
{
  "orderNo": "YX20260610A001",
  "totalFee": 299900,
  "body": "ThinkPad X1 Carbon i7-1355U...",
  "openid": "oGZUI0eg********",
  "attach": "订单附加数据"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `orderNo` | string | ✅ | ERP 订单号，用于 `out_trade_no` |
| `totalFee` | integer | ✅ | 支付金额，单位：**分**（如 ¥2999.00 传 `299900`） |
| `body` | string | ✅ | 商品描述，会显示在用户微信支付账单中 |
| `openid` | string | ✅ | 用户在该小程序的 openid |
| `attach` | string | ❌ | 附加数据，支付成功通知时原样返回 |

### 后端内部调用流程

#### 步骤 1：调用微信统一下单接口（API V3）

```
POST https://api.mch.weixin.qq.com/v3/pay/transactions/jsapi

Header:
  Authorization: WECHATPAY2-SHA256-RSA2048 mchid="1746811683",nonce_str="...",signature="...",timestamp="...",serial_no="..."
  Content-Type: application/json
  Accept: application/json

Body:
{
  "appid": "wx8cf5c2ef49e867fe",
  "mchid": "1746811683",
  "description": "{body}",
  "out_trade_no": "{orderNo}",
  "notify_url": "https://erp.qixiangshou.com/api/v1/payment/notify",
  "amount": {
    "total": {totalFee},
    "currency": "CNY"
  },
  "payer": {
    "openid": "{openid}"
  },
  "attach": "{attach}"
}
```

> ⚠️ **签名认证**（WECHATPAY2-SHA256-RSA2048）需要商户 API 私钥，参考：[微信支付签名文档](https://pay.weixin.qq.com/doc/v3/merchant/4012365342)

微信返回：
```json
{
  "prepay_id": "wx201410272009395522657a690389285100"
}
```

#### 步骤 2：生成小程序调起支付参数

对以下参数使用商户私钥签名：

```
appId:     "wx8cf5c2ef49e867fe"
timeStamp: "1718000000"  (String 时间戳，秒)
nonceStr:  "abc123..."   (随机字符串，32位以内)
package:   "prepay_id=wx201410272009395522657a690389285100"
signType:  "RSA"
paySign:   (用商户私钥对以上字段按字典序拼接后 RSA 签名)
```

> **MD5 签名方式（备选）**：按 `appId=xxx&nonceStr=xxx&package=prepay_id=xxx&signType=MD5&timeStamp=xxx&key={商户API密钥}` 拼接后 MD5

### 响应体

```json
{
  "prepay_id": "wx201410272009395522657a690389285100",
  "timeStamp": "1718000000",
  "nonceStr": "abc123def456",
  "package": "prepay_id=wx201410272009395522657a690389285100",
  "signType": "RSA",
  "paySign": "U2lnbmF0dXJl..."
}
```

前端拿到这些参数后直接调用 `wx.requestPayment()` 拉起微信支付。

### 错误响应

```json
{
  "errcode": "ORDER_EXISTS",
  "errmsg": "订单已存在，请勿重复支付"
}
```

## 接口四：支付结果通知（已存在，补充规范）

### 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/payment/notify` |
| 鉴权 | 微信支付签名验证 |
| 用途 | 微信支付异步通知回调地址 |

### 微信 POST 过来的 Body

```json
{
  "id": "EV-2018022511223320873",
  "create_time": "2015-05-20T13:29:35+08:00",
  "resource_type": "encrypt-resource",
  "event_type": "TRANSACTION.SUCCESS",
  "summary": "支付成功",
  "resource": {
    "original_type": "transaction",
    "algorithm": "AEAD_AES_256_GCM",
    "ciphertext": "...",
    "associated_data": "",
    "nonce": "..."
  }
}
```

### 后端需做的事

1. 验证签名（使用微信支付平台公钥）
2. 解密 `resource.ciphertext` 得到交易详情
3. 根据 `out_trade_no` 找到对应订单，更新订单状态为 "已支付"
4. 返回 `{ "code": "SUCCESS", "message": "成功" }` 给微信

## 商户 API 证书配置

### 证书文件清单

从商户平台下载的证书包包含以下文件：

| 文件 | 用途 | 放置位置 |
|------|------|----------|
| `apiclient_cert.pem` | 商户证书（公钥） | ERP 服务器配置目录 |
| `apiclient_key.pem` | 商户 API 私钥 | ERP 服务器配置目录（**严格保密**） |
| `apiclient_cert.p12` | PKCS12 格式证书 | 备用（某些 SDK 需要） |

### 证书信息

| 项目 | 内容 |
|------|------|
| 商户号 (mchid) | `1746811683` |
| 证书序列号 | `3D3E9915F42FD81C5373C07891B2D0C15573A1B3` |
| 颁发时间 | 2026-06-10 |
| 到期时间 | 2031-06-09（5 年有效） |

### ERP 后端使用方式

后端支付统一下单接口需要用 `apiclient_key.pem`（私钥）对请求做 SHA256-RSA2048 签名。

**签名生成逻辑（伪代码）：**

```
method = "POST"
url = "/v3/pay/transactions/jsapi"
timestamp = String(Math.floor(Date.now() / 1000))
nonce_str = randomString(32)
body = JSON.stringify(requestBody)

message = method + "\n" + url + "\n" + timestamp + "\n" + nonce_str + "\n" + body + "\n"
signature = RSA_SHA256_sign(message, apiclient_key.pem)

Authorization = "WECHATPAY2-SHA256-RSA2048 " +
  "mchid=\"1746811683\"," +
  "nonce_str=\"" + nonce_str + "\"," +
  "signature=\"" + signature + "\"," +
  "timestamp=\"" + timestamp + "\"," +
  "serial_no=\"3D3E9915F42FD81C5373C07891B2D0C15573A1B3\""
```

### 环境变量建议

ERP 服务端建议通过环境变量注入（不要硬编码）：

```bash
WX_APP_ID=wx8cf5c2ef49e867fe
WX_APP_SECRET=<小程序AppSecret>
WX_MCH_ID=1746811683
WX_MCH_CERT_PATH=/etc/ssl/apiclient_cert.pem
WX_MCH_KEY_PATH=/etc/ssl/apiclient_key.pem
WX_MCH_CERT_SERIAL=3D3E9915F42FD81C5373C07891B2D0C15573A1B3
WX_PAY_NOTIFY_URL=https://erp.qixiangshou.com/api/v1/payment/notify
```

### 需要在微信平台配置的内容

| 配置项 | 路径 | 说明 |
|--------|------|------|
| 小程序 AppSecret | 微信公众平台 → 开发 → 开发设置 | 用于 code2Session |
| 商户 API 私钥/证书 | 微信支付商户平台 → API 安全 | 用于支付签名 |
| 支付回调域名 | 微信支付商户平台 → 开发配置 | 设置 `erp.qixiangshou.com` |
| 服务器域名白名单 | 微信公众平台 → 开发 → 服务器域名 | request 合法域名添加 `erp.qixiangshou.com` |
